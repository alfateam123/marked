[small]Getting the Gist of Markdown's Formatting Syntax[/small] This page offers a brief overview of what it's like to use Markdown. The [url="/projects/markdown/syntax"]syntax page[/url] provides complete, detailed documentation for every feature, but Markdown should be very easy to pick up simply by looking at a few examples of it in action. The examples on this page are written in a before/after style, showing example syntax and the HTML output produced by Markdown. It's also helpful to simply try Markdown out; the [url="/projects/markdown/dingus"]Dingus[/url] is a web application that allows you type your own Markdown-formatted text and translate it to XHTML. [b]Note:[/b] This document is itself written using Markdown; you can [url="/projects/markdown/basics.text"]see the source for it by adding '.text' to the URL[/url]. [small]Paragraphs, Headers, Blockquotes[/small] A paragraph is simply one or more consecutive lines of text, separated by one or more blank lines. (A blank line is any line that looks like a blank line -- a line containing nothing spaces or tabs is considered blank.) Normal paragraphs should not be intended with spaces or tabs. Markdown offers two styles of headers: [i]Setext[/i] and [i]atx[/i]. Setext-style headers for [code=text]<h1>[/code] and [code=text]<h2>[/code] are created by "underlining" with equal signs ([code=text]=[/code]) and hyphens ([code=text]-[/code]), respectively. To create an atx-style header, you put 1-6 hash marks ([code=text]#[/code]) at the beginning of the line -- the number of hashes equals the resulting HTML header level. Blockquotes are indicated using email-style '[code=text]>[/code]' angle brackets. Markdown: [code=text]A First Level Header ==================== A Second Level Header --------------------- Now is the time for all good men to come to the aid of their country. This is just a regular paragraph. The quick brown fox jumped over the lazy dog's back. ### Header 3 > This is a blockquote. > > This is the second paragraph in the blockquote. > > ## This is an H2 in a blockquote [/code] Output: [code=text]<h1>A First Level Header</h1> <h2>A Second Level Header</h2> <p>Now is the time for all good men to come to the aid of their country. This is just a regular paragraph.</p> <p>The quick brown fox jumped over the lazy dog's back.</p> <h3>Header 3</h3> <blockquote> <p>This is a blockquote.</p> <p>This is the second paragraph in the blockquote.</p> <h2>This is an H2 in a blockquote</h2> </blockquote> [/code] [small]Phrase Emphasis[/small] Markdown uses asterisks and underscores to indicate spans of emphasis. Markdown: [code=text]Some of these words *are emphasized*. Some of these words _are emphasized also_. Use two asterisks for **strong emphasis**. Or, if you prefer, __use two underscores instead__. [/code] Output: [code=text]<p>Some of these words <em>are emphasized</em>. Some of these words <em>are emphasized also</em>.</p> <p>Use two asterisks for <strong>strong emphasis</strong>. Or, if you prefer, <strong>use two underscores instead</strong>.</p> [/code] [small]Lists[/small] Unordered (bulleted) lists use asterisks, pluses, and hyphens ([code=text]*[/code], [code=text]+[/code], and [code=text]-[/code]) as list markers. These three markers are interchangable; this: [code=text]* Candy. * Gum. * Booze. [/code] this: [code=text]+ Candy. + Gum. + Booze. [/code] and this: [code=text]- Candy. - Gum. - Booze. [/code] all produce the same output: [code=text]<ul> <li>Candy.</li> <li>Gum.</li> <li>Booze.</li> </ul> [/code] Ordered (numbered) lists use regular numbers, followed by periods, as list markers: [code=text]1. Red 2. Green 3. Blue [/code] Output: [code=text]<ol> <li>Red</li> <li>Green</li> <li>Blue</li> </ol> [/code] If you put blank lines between items, you'll get [code=text]<p>[/code] tags for the list item text. You can create multi-paragraph list items by indenting the paragraphs by 4 spaces or 1 tab: [code=text]* A list item. With multiple paragraphs. * Another item in the list. [/code] Output: [code=text]<ul> <li><p>A list item.</p> <p>With multiple paragraphs.</p></li> <li><p>Another item in the list.</p></li> </ul> [/code] [small]Links[/small] Markdown supports two styles for creating links: [i]inline[/i] and [i]reference[/i]. With both styles, you use square brackets to delimit the text you want to turn into a link. Inline-style links use parentheses immediately after the link text. For example: [code=text]This is an [example link](http://example.com/). [/code] Output: [code=text]<p>This is an <a href="http://example.com/"> example link</a>.</p> [/code] Optionally, you may include a title attribute in the parentheses: [code=text]This is an [example link](http://example.com/ "With a Title"). [/code] Output: [code=text]<p>This is an <a href="http://example.com/" title="With a Title"> example link</a>.</p> [/code] Reference-style links allow you to refer to your links by names, which you define elsewhere in your document: [code=text]I get 10 times more traffic from [Google][1] than from [Yahoo][2] or [MSN][3]. [1]: http://google.com/ "Google" [2]: http://search.yahoo.com/ "Yahoo Search" [3]: http://search.msn.com/ "MSN Search" [/code] Output: [code=text]<p>I get 10 times more traffic from <a href="http://google.com/" title="Google">Google</a> than from <a href="http://search.yahoo.com/" title="Yahoo Search">Yahoo</a> or <a href="http://search.msn.com/" title="MSN Search">MSN</a>.</p> [/code] The title attribute is optional. Link names may contain letters, numbers and spaces, but are [i]not[/i] case sensitive: [code=text]I start my morning with a cup of coffee and [The New York Times][NY Times]. [ny times]: http://www.nytimes.com/ [/code] Output: [code=text]<p>I start my morning with a cup of coffee and <a href="http://www.nytimes.com/">The New York Times</a>.</p> [/code] [small]Images[/small] Image syntax is very much like link syntax. Inline (titles are optional): [code=text]![alt text](/path/to/img.jpg "Title") [/code] Reference-style: [code=text]![alt text][id] [id]: /path/to/img.jpg "Title" [/code] Both of the above examples produce the same output: [code=text]<img src="/path/to/img.jpg" alt="alt text" title="Title" /> [/code] [small]Code[/small] In a regular paragraph, you can create code span by wrapping text in backtick quotes. Any ampersands ([code=text]&[/code]) and angle brackets ([code=text]<[/code] or [code=text]>[/code]) will automatically be translated into HTML entities. This makes it easy to use Markdown to write about HTML example code: [code=text]I strongly recommend against using any `<blink>` tags. I wish SmartyPants used named entities like `&mdash;` instead of decimal-encoded entites like `&#8212;`. [/code] Output: [code=text]<p>I strongly recommend against using any <code>&lt;blink&gt;</code> tags.</p> <p>I wish SmartyPants used named entities like <code>&amp;mdash;</code> instead of decimal-encoded entites like <code>&amp;#8212;</code>.</p> [/code] To specify an entire block of pre-formatted code, indent every line of the block by 4 spaces or 1 tab. Just like with code spans, [code=text]&[/code], [code=text]<[/code], and [code=text]>[/code] characters will be escaped automatically. Markdown: [code=text]If you want your page to validate under XHTML 1.0 Strict, you've got to put paragraph tags in your blockquotes: <blockquote> <p>For example.</p> </blockquote> [/code] Output: [code=text]<p>If you want your page to validate under XHTML 1.0 Strict, you've got to put paragraph tags in your blockquotes:</p> <pre><code>&lt;blockquote&gt; &lt;p&gt;For example.&lt;/p&gt; &lt;/blockquote&gt; </code></pre> [/code]
